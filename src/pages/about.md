你好，我是 **52coder**。

这里是我的技术笔记本 —— 记录那些「查资料花了三小时、写下来只要二十分钟」的东西。比起罗列结论，我更愿意把**排查过程、踩过的坑和最终的取证**一并写下来。因为半年后回头看，真正有价值的往往是当时怎么一步步想明白的，而不是结论本身。

## 在写些什么

**C/C++ 与系统编程** —— 主要方向。写过 [协程](/posts/coroutine/)、[C++ 模板使用例子](/posts/cpp_template/)、[设计模式的 C++17 实现](/posts/cpp17_design_patterns/)、[abseil C++ Tips](/posts/abseil_cpp_tips/)，也记录过 [vector 动态扩容存在的陷阱](/posts/vector_growth/) 这类容易忽略的细节。

**源码阅读** —— 读源码是理解一个系统最快的路径。写过 [leveldb 源码解析](/posts/leveldb_source/)、[从零搭建 C++ 推荐服务框架](/posts/rank-framework-build-journey/)，也拆过 [Claude Code 的源码架构](/posts/claude-code-code-review-plugin-source-analysis/)。

**量化交易** —— 近期在探索的新方向，从零开始的完整记录：[基础知识](/posts/quant-basics-1-stock-fundamentals/)、[macOS 搭建 AKShare 数据环境](/posts/quant-tools-1-akshare-macos-setup/)、[vnpy 回测框架接入海龟策略](/posts/quant-tools-2-vnpy-turtle-notebook/)。

**工具与工程实践** —— [MySQL 基础知识](/posts/mysql-easy/)、[Ghostty + Yazi 现代终端配置](/posts/ghostty-yazi-terminal-setup/)、以及[这个博客本身的搭建与部署](/posts/astropaper-local-build-and-vercel-deploy/)。
